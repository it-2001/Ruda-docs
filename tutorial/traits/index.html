Traits

Traits are a way to define a set of methods that a type can implement. Traits are similar to interfaces in other languages.

Example:

trait Animal {
    fun speak()
}

struct Dog {
    name: string,
}

impl Dog {
    constructor(name: string) {
        this.name = name
    }
}

impl Dog trait Animal {
    fun speak() {
        io.println("Woof!")
    }
}

fun main() {
    let dog = Dog("Rover")
    dog.speak()
}

Output:

Woof!

In this example, we define a trait Animal that has a method speak. We then define a struct Dog that implements the Animal trait. This means that the Dog struct must have a method speak that matches the signature of the speak method in the Animal trait.

We can then create a Dog struct and call the speak method on it.

Trait Constraints

Traits can be used as constraints on type parameters.

Example:

struct Pair<T(Animal), U(Animal)> {
    first: T,
    second: U,
}

fun main() {
    let pair = Pair(Dog("Rover"), Cat("Spot"))
    pair.first.speak()
    pair.second.speak()
}

Output:

Woof!
Meow!

In this example, the Pair struct has two type parameters: T and U. These type parameters must implement the Animal trait. This means that the Pair struct can only be used with types that implement the Animal trait.

Here we can call the speak method on the first and second fields of the pair variable because they are both of type Dog and Cat which implement the Animal trait.
