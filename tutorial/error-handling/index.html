Error handling

Sometimes an operation can fail. For example, if you try to open a file that doesn't exist, or if you try to divide by zero. In Ruda, you can handle these errors using the try keyword.

try {
    let file = io.open("file.txt")
} catch err {
    io.println(err)
}

The try keyword is used to execute a block of code that may fail. If the block of code fails, the catch keyword is used to execute a block of code that handles the error.

Catch statements

Catch statements are used to handle errors. They are similar to if statements, but they are used to handle errors instead of boolean values.

try {
    let file = io.open("file.txt")
} catch err {
    io.println(err)
}

They can also be used to handle specific errors.

try {
    let file = io.open("file.txt")
} catch err: io.FileNotFoundError {
    io.println("file not found")
} catch err: io.FileAccessError {
    io.println("file access error")
} catch err {
    io.println(err.message())
}

Yeet

The yeet keyword is used to throw an error.

yeet Error("something went wrong")

This is useful for creating your own errors.

fun divide(x: int, y: int)!: int {
    if y == 0 {
        yeet Error("cannot divide by zero")
    }

    return x / y
}

On the first line we use the ! operator to tell the compiler that this function can fail. Then we use the yeet keyword to throw an error if the divisor is zero.

Bang operator

If we know that a function can't fail, we can use the ! operator to tell the compiler that it can't fail.

let x = divide(5, 5)!

If the function does fail it will yeet the error to caller of the function.

let x = divide(5, 0)!

If you never properly handle the error, the program will crash.

All errors must be handled either by using the try keyword or the ! operator. If not, the program won't compile.

Error declarations

You can declare your own errors using the error keyword.

error DivisionByZeroError(number: int) {
    message: "cannot divide by zero",
    code: 1,
}

This error can be used like this:

fun divide(x: int, y: int)!: int {
    if y == 0 {
        yeet DivisionByZeroError(x)
    }

    return x / y
}

The error field code is optional. It can be used to identify the error in system. If it is not specified, it will default to 1.

You can also take message as an argument.

error Error(message: string?) {
    message: {
        if message? {
            return message
        } else {
            return "something went wrong"
        }
    }
    code: 1,
}
